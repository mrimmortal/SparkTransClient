# Feature Workflow

Use when the repo needs a repeated feature-development workflow.

## Standard flow

1. Read `AGENTS.md`.
2. Read `AI_CONTEXT.md`.
3. Read `docs/MODULE_MAP.md`.
4. Read relevant `.ai-rules/*.md`.
5. Inspect only relevant source files.
6. Make the smallest safe change.
7. Add/update focused tests when behavior changes.
8. Run validation from `docs/COMMANDS.md`.
9. Update docs only if behavior, architecture, config, commands, public contracts, or operational workflow changed.
10. Add a compact `docs/SESSION_LOG.md` entry after meaningful work.
