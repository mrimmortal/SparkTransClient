# Final Response

Use this before responding after code/documentation work.

## Required format

End with:

1. Summary
2. Changed files
3. Validation run with pass/fail/not-run status
4. Whether AI context files were updated or not needed
5. Risks, assumptions, or follow-ups, only if any

## Rules

- Be concise.
- Be factual.
- Be practical.
- Do not over-explain.
- Do not claim unverified success.
- Do not claim tests passed unless they actually ran and passed.
- If validation was not run, say why.
- If something is unknown, say `Unknown` or `To verify`.
